# 项目 Agent 规则

## 对话记录

后续任何 assistant/agent 在本项目中处理用户请求时，都必须在最终回复前更新 `docs/conversation-log.md`。只要本轮包含有意义的用户请求、实现、命令执行、设计决策或文件修改，就需要追加记录。

每条记录必须包含：

- 对话产生时间，格式建议为 `YYYY-MM-DD HH:mm:ss 时区`。
- 用户提示词，必须完整保留原文。
- 回复或执行动作的简短中文概述。
- 创建或修改过的文件，使用仓库相对路径。
- 执行过的验证命令，如有。

所有对话记录内容必须使用中文。代码、命令、文件路径、仓库地址以及用户原始提示词中的非中文内容可以保持原样。

如果某一轮只是普通交流且没有文件修改，也要追加用户原始提示词和中文回复简述，并写明“修改文件：无”。

这是一条项目级长期规则。用户后续不需要再次提醒。
